視差捲動效果

作者：Vineet Kumar

什麼是視差捲動？

視差捲動是一種網頁設計技巧，會讓背景以比前景更慢的速度移動。 當訪客往下捲動時，畫面會呈現 3D 深度感並帶來更沉浸的瀏覽體驗。 這項效果利用了視覺上的錯覺：人眼會把距離較近的物體看得比遠處物體大， 因此遠處看起來移動得更慢。這種錯覺早已被運用在不同媒介上，打造逼真的效果。 最早可以追溯到迪士尼《白雪公主與七矮人》等傳統動畫，以及《超級瑪利歐》等早期電玩。 隨著 CSS 與 HTML 的進步，視差效果才演變成今日在網頁設計上的運用。

早期在瀏覽器實作視差捲動的案例之一，是 2007 年開發者 Glutnix 在部落格上分享的範例， 透過 JavaScript 與 CSS2 展示，並支援當時的 Internet Explorer 6 等瀏覽器。 直到 2011 年 HTML5 與 CSS3 推出後，這項技術才在設計師間流行起來。 支持者認為這是擁抱網頁流動性的簡單方式，並利用視差背景提升互動性與整體體驗。 然而，普渡大學在 2013 年發表的研究指出：「雖然視差捲動提升了某些使用者體驗面向， 但不一定能改善整體的使用者體驗。」


為什麼要使用視差圖像？

    設計師嘗試視差圖像有兩個主要原因：

  • 更有故事性

    視差圖像非常適合說故事。優秀的設計就是溝通，打造網站時我們希望與訪客對話， 並講述關於產品、服務或品牌的動人故事。視差捲動能讓訪客沉浸在獨特的線上體驗， 甚至可以在單頁網站中呈現完整故事，讓使用者不必跳轉頁面也能一路看完。

  • 提升使用者互動

    視差圖像也能提升互動性。降低跳出率是網頁設計的重要目標之一， 跳出率代表只看一頁就離開的訪客比例，數值越高越會影響搜尋排名。 精心設計的視差頁面能帶來動態體驗，讓使用者更願意停留。


如果想在網站加入視差捲動效果，可以參考以下建議：

  • 適度運用

    與其處處使用，不如把視差效果放在特定元素上會更有效果。 可以用在標題、頁首或僅限首頁，重點是強化設計而不是分散注意力。

  • 圖像壓縮

    視差效果需要各種媒體與 CSS 形狀來營造深度感。 透過壓縮圖片避免載入過慢，才能兼顧效能與體驗。

  • 運用色彩

    視差不只關於前景與背景，也包含色彩搭配。 背景可以使用較低飽和度的色調，前景則用更鮮明的顏色，讓層次感更突出。
